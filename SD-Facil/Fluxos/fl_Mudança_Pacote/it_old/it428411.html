NETSMS/NETSALES – Mudança de pacote
(Mais de uma vez no mesmo dia e no mesmo ponto)

RESUMO:

Orientar os analistas sobre como tratar os casos onde o ocorre o erro ao gerar mais de uma solicitação de Mudança de Pacote no mesmo dia.

Aplicação: NETSMS/NETSALES

Fluxo de atividades:

Para o problema temos o seguinte cenário mapeado :

Usuarios realizam a tentativa de abrir a solicitação MUDANCA DE PACOTE. Porem é exibida a mensagem “Não é permitido a mudança de pacote para o mesmo produto no mesmo ponto, duas vezes no mesmo dia ”, devido regra de negocios NET, onde só é permitida uma MUDANÇA de PACOTE para o mesmo produto e ponto por dia.

cenário:

Usuário reporta que ocorre erro a Mensagem “Não é permitido a mudança de pacote para o mesmo produto no mesmo ponto, duas vezes no mesmo dia ” ao gerar a solicitação de Mudança de Pacote.

Exemplo:

Caso o contrato se enquadre no cenario acima onde o usuario tenta abrir mais de uma mudança de pacote, para o ponto no mesmo dia. O analista NÃO DEVERÁ abrir chamado para as filas HP, Deve-se orientar o usuario a observar no historico de solicitação do contrato que ja foi aberta esta mesma mudança de pacote no dia atual e que o mesmo prossiga com a mudança de pacote somente no dia seguinte.

Resposta Padrão Sugerida:

“SEGUNDO REGRA DE NEGÓCIO DA NET NÃO É PERMITIDO GERAR A MESMA MUDANÇA DE PACOTE NO MESMO PONTO, MAIS QUE UMA VEZ AO DIA. FAVOR PROSSEGUIR COM A SOLICITAÇÃO DESEJADA NO DIA SEGUINTE."
