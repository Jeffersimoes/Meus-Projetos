Validar permissão para abertura de solicitação - Mudança de Pacote

IT:431199

NETSMS - Consultar permissão para gerar solicitação (mudança de pacote)

RESUMO:

Script para verificar se usuário possui permissão para gerar/abrir a solicitação desejada, para este script abertura da solicitação de mudança de pacote.

PROBLEMA:

Validação de permissão de abertura solicitação mudança de pacote.

Identificação do cenário:

Verificar se produto desejado aparece na lista na abertura da solicitação de Mudança de Pacote

Exemplo:

RESOLUÇÃO:

1. Consultar permissão para abertura da solicitação de MUDANCA DE PACOTE.

  • - Acessar o HP Business Tool: Clique aqui

  • Caso apareça a seguinte mensagem: ‘There is a problem with this website's security certificate.’
    Clique em ‘Continue to this website (not recommended)’ para prosseguir

  • - Logar com mesmo usuário utilizado para acessar o NETSMS

  • - Na opção de CONSULTAR AÇÃO, preencher a respectiva cidade ao qual o usuário esta acessando no sistema e os demais filtros da seguinte forma:
    Grupo : CRM
    Macro Processo : CRM / Acesso / Perfil
    Funcionalidade : Perfil / Usuario
    Ação : CONSULTAR PERMISSÃO PARA GERAR SOLIC

    - Preencher os campos conforme descrito:
    Cod operadora: preencher com o códido da operadora desejada. Ex: 003
    Login: usuário utilizado durante a tentativa de geração da solic. Ex: T665777
    Id solicitação: o id da solicitação desejada obtido através da funcionalidade CRM/ATENDER – CONTRATO/SOLICITACAO – CONSULTAR ID SOLICITAÇÃO
    Produto desejado: o id do produto relacionado a solicitação desejada.
    Produto atual: o produto atualmente instalado no ponto.
    Importante: Caso o usuário deseje realizar uma adesão de ponto ou contratação de a la carte então este campo deverá ser preenchido com o número 0.

Caso o usuário não possua permissão, encerrar o incidente no Service DesK N1

OBS: Este procedimento é apenas para validação de permissão.
