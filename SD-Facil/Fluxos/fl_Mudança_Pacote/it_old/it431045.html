NETSMS/NETSALES – Mudança de pacote (perfil e comercialização de produto)

RESUMO:

Verificar se produto pertence ao perfil desejado e se sua comercialização encontra-se aberta.

PROBLEMA:

PERFIL, PRODUTO, COMERCIALIZACAO, PERFIL DO CONTRATO, PRODUTO PERTENCE AO COMBO.

Aplicação: NETSMS/NETSALES

Script para verificar se produto desejado encontra-se disponível para o perfil X e se sua comercialização está aberta.

A abertura de chamado deverá ser feita apenas se mesmo após a validação o produto ainda não aparece disponível para contratação.

Exemplo:

Usuário reporta que ocorre erro ao gerar solicitação de adesão/mudança de pacote.

VALIDAÇÃO:

1. Consultas na ferramenta HP Business Tool.

  • - Acessar o HP Business Tool: Clique aqui

  • Caso apareça a seguinte mensagem: ‘There is a problem with this website's security certificate.’
    Clique em ‘Continue to this website (not recommended)’ para prosseguir

  • - Logar com mesmo usuário utilizado para acessar o NETSMS

  • - Na opção de CONSULTAR AÇÃO, preencher a respectiva cidade ao qual o usuário esta acessando no sistema e os demais filtros da seguinte forma:
    Grupo : CRM
    Macro Processo : CRM / Vender / Venda
    Funcionalidade : Venda / Produto
    Ação : CONSULTA PERFIL E COMERCIALIZACAO DE PRODUTO

    - Preencher os campos conforme descrito:
    Cod operadora: preencher com o códido da operadora desejada. Ex: 003
    Perfil desejado: perfil ao qual usuário deseja contratar o produto. Ex COMBO 1244 / PTV VIRTUA FONE
    Produto desejado: ID do produto desejado para contratação
    Produto atual: o produto atualmente instalado no ponto.
    Importante: Caso o usuário deseje realizar uma adesão de ponto ou contratação de a la carte, este campo deverá ser preenchido com o número 0.

Caso a validação informe que o produto esta disponivel para o perfil correto que o usuario esta tentando realizar a contratação e mesmo assim não aparecer, ou seja tem que selecionar o perfil correto que a funcionalidade acima informar que o perfil pertença, independente do fique ligado, visto que a funcionalidade é uma consulta real time da base.

Caso mesmo após todas as validações necessárias, ainda não for possível identificar o motivo da falha, abrir incidente conforme informações abaixo:
