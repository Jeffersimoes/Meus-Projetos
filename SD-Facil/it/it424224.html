NETSMS - Falha: 'Não foi encontrado registro de telefone VoIP para ser cancelado'

RESUMO:

Esse procedimento tem o objetivo de orientar o colaborador como deverá agir caso apresente erro no ponto VOIP.

APLICAÇÃO:

Essa IT possuí aplicabilidade nos seguintes grupos:

• Service Desk N1

DESCRIÇÃO GERAL DO PROCEDIMENTO:

Existem duas situações que podem apresentar erro no VOIP

'Não foi encontrado registro de Telefone VoIP para ser cancelado'

O analista deverá seguir o procedimento abaixo para validar se usuário anexou corretamente o passo a passo até o erro.
Acessar contrato no sistema NETSMS > Aba Produtos > Abrir a solicitação > Dados da OS > Baixa de OS.

O erro ocorre durante o processo de baixa da OS.

Segue o passo a passo com os respectivos exemplos:

1.1 Abra a aba produtos e busque a solicitação referente a ponto de instalação do Voip.

1.2 Posteriormente deverá verificar se a OS referenciada esta com status em aberto.

1.3 No painel de ferramentas o usuário deverá acessar Rede Interna>OS’s>Baixa de OS, e realizar a baixa da OS manualmente.

1.4 O analista deverá validar se o usuário tentou EXECUTAR a OS, conforme mostra imagem do erro no processo de execução.

1.5 Será necessário a verificação se o usuário tentou CANCELAR a OS, pois é necessário possuir as duas print’s do erro para dar andamento no chamado

Após realizar todas as validações e se o chamado estiver com os anexos dos erros de EXECUÇÃO e CANCELAMENTO o analista deverá seguir com a solução.

SOLUÇÃO:

Chamados com erro VOIP, deveram ser categorizados e encaminhados para NETSMS – Portabilidade.
Colaborador deverá preencher corretamente, aba portabilidade corretamente.

Categoria: SISTEMAS.NETSMS.SOLICITACOES.EXISTENCIA DE OS S DEPENDENTES EM ANDAMENTO SENDO TRATADAS PELA REDE INTERNA

Categoria: SISTEMAS.NETSMS.PORTABILIDADE - TELA DE INF VOIP.O C TROCA DE NUM EXECUTADA-Ñ OCORREU TROCA NA TELA DE INF VOIP

Categoria: SISTEMAS.NETSMS.PORTABILIDADE - TELA DE INF VOIP.NAO APARECE DADOS NETFONE

Todas essas categorias serão direcionadas para o Grupo Responsável: NETSMS - PORTABILIDADE


SUGESTÃO DE RESPOSTA:

Prezados,
Após análise do chamado, e conforme procedimento, somos orientados a transferir o chamado para equipe de NETSMS – Portabilidade, favor dar andamento com a tratativa.

Atenciosamente.
