Validar permissão para abertura de solicitação - Mudança de Pacote


OBJETIVO:

Esse procedimento tem o objetivo orientar o analista como validar se o usuário possui permissões para gerar/abrir solicitação.


APLICAÇÃO:

Service Desk N1


REFERÊNCIAS:

N/A


DESCRIÇÃO GERAL DO PROCEDIMENTO

Primeiro passo o analista deverá acessar o site do HPBT.

HP BusinessTool

Caso o analista não saiba o número do id da solicitação segue a macro para consultar o ID.

  • Grupo: CRM
  • Macro Processo: CRM/Atender
  • Funcionalidade: Contrato/ Solicitação
  • Ação: Consultar ID Solicitação.

Após identificar a ID do tipo de solicitação desejada, o analista deverá consultar se o usuário possui permissão para gerar esta solicitação. Esta validação é realizada por meio da macro abaixo:

  • Grupo: CRM
  • Macro Processo: CRM / Acesso / Perfil
  • Funcionalidade: Perfil / Usuario
  • Ação: CONSULTAR PERMISSÃO PARA GERAR SOLICITAÇÃO ou Permissão para Gerar Solic

Segue o resultado da validação, se o usuário possui PERMISSÃO PARA GERAR SOLICITAÇÃO




SOLUÇÃO:

Caso o usuário não possua permissão o chamado deverá ser encerrado em N1.

Se o usuário possuir a permissão o analista deverá seguir com as validações conforme o problema.

Este procedimento tem o objetivo somente de validar as permissões do usuário.


SUGESTÃO DE RESPOSTA:


Prezado usuário,

Durante analise, foi identificado que o login informado não possui o permissão para gerar a solicitação desejada.
Esta solicitação deve ser gerada por um usuário que possua as permissões necessárias.
Caso o colaborador necessite deste perfil, será necessario entrar em contato com sua supervisão para que seja aberta uma solicitação de permissão de acesso.

Atenciosamente,
