Baixa de OS com erro: FORMATO DE DATA MUITO LONGO PARA BUFFER INTERNO


OBJETIVO:

Esse procedimento visa orientar o analista a identificar a falha de baixa de OS com erro de Formato de Data muito longo para Buffer interno.


APLICAÇÃO:

Essa IT possuí aplicabilidade nos seguintes grupos:

SERVICE DESK N1

Netsms – Instalar


DESCRIÇÃO GERAL DO PROCEDIMENTO:

Chamados onde o usuário informa erro ao baixar OS conforme imagem abaixo, deverá ser direcionado conforme instrução no item solução.


SOLUÇÃO:

Direcionar o incidente conforme instrução abaixo, preencher a aba propriedade e colocar os itens abaixo:

  • Login do usuário para baixa da OS:
  • Data e hora de início da execução:
  • Data e hora de término da execução:
  • Tipo de Fechamento (Executado / Cancelado):
  • Imediatada? (Sim / Não):

Utilizar a seguinte categoria para direcionamento:

Classificação: Incidente - A3

Categoria: SISTEMAS.NETSMS.OS.OS PROCESSADA E Ñ BAIXADA

Grupo: NETSMS – INSTALAR


SUGESTÃO DE RESPOSTA:

Prezados,

Após análise do chamado, identificamos que se trata de uma falha Sistêmica no sistema NETSMS.

Foi confirmado que o usuário possui todas as permissões necessárias para realizar a ação desejada, conforme evidencia em anexo. Contudo, devido ao erro informado não foi possível o usuário prosseguir com a solicitação de Baixa de OS.

Com base nas informações apresentadas e na IT 424618 o chamado deverá ser tratado pela equipe NETSMS – INSTALAR.

Sendo assim, solicitamos por gentileza que prossigam com as tratativas necessárias para resolução do chamado.

Atenciosamente,
Service Desk Claro Brasil
