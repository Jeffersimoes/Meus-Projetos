Validações para movimentação de terminal (Perda ou Roubo)


OBJETIVO:

Esse procedimento tem o objetivo orientar o analista como fazer as validações necessárias para direcionamento ou encerramento.


APLICAÇÃO:

Essa IT possuí aplicabilidade nos seguintes grupos:

  • Service Desk N1
  • Atlas

REFERÊNCIAS:

N/A


DESCRIÇÃO GERAL DO PROCEDIMENTO

O analista deverá solicitar evidência com o status do terminal.

  • Perda
  • Roubo
  • Desaparecido
  • O status do terminal é perda/roubo/desaparecido?

Se a resposta for SIM o analista deverá questionar se o usuário já entrou em contato com MSO.
Nesse caso o chamado deverá ficar pendente usuário aguardando retorno, se contato for via e - mail o
mesmo deverá incluso no chamado. Se a reposta for Não, o analista deverá seguir para a próxima validação.

Validar se apresenta os erros citados. Caso apresente o erro, o analista deverá encerrar o chamado seguindo sugestão de resposta 7.

Caso apresente outro erro seguir com as validações abaixo.


Cenário 2 – Erro CRM-0268

Validar se apresenta os erros citados. Caso apresente o erro, o analista deverá encerrar o chamado seguindo sugestão de resposta 7..

Validar se apresenta os erros citados. Caso apresente o erro, o analista deverá encerrar o chamado seguindo sugestão de resposta 7..

Cenário 4 - Procedimento para ser feito a Entrega/Devolução do terminal

No Netsms o usuário deverá acessar Menu de Opções > Rede Interna -> OS's -> Supervisão de Rede Interna -> Supervisão de Rede Interna-Principal


Cenário 5 – Problema ainda persiste e foi realizado procedimento no cenário 4

Nesse caso o chamado deverá ser encaminha para equipe de atlas, seguindo a categoria disponibilizada no item 6 Solução.

Para o cenário 5 o analista deverá categorizar conforme a descrição do incidente.

  • SISTEMAS.NETSMS.TERMINAIS DEVOLUCAO.TELA DE SUPERVISAO DE REDE INTERNA

Grupo solucionador: Atlas


SUGESTÃO DE RESPOSTA:

Cenário 1 - Erro ORA-01858 - ORA-06512

Prezado(a),

Esse erro ocorre por um problema local na máquina, especificamente na configuração de data/hora.
Favor seguir o procedimento abaixo e ajustar data/hora:
Painel de controle >> opções regionais de idioma >> personalizar >> data >> a data abreviada deve ser modificada para dd/mm/aaaa.
Caso o problema ocorra novamente após o erro reabrir o chamado anexando novas evidências para análise.

Atenciosamente,
Service Desk Claro Brasil


Cenário 2 – Erro CRM-0268

Prezado(a),

De acordo com as regras de negócios da Net, é necessário que um ponto com pacote VIRTUA possua contratações a La carte do provedor e/ou forma de aquisição.
Solução: Para o usuário conseguir efetuar qualquer alteração neste ponto, o mesmo terá que efetuar a contratação destes a La cartes para acerto sistêmico.

Atenciosamente,
Service Desk Claro Brasil


Cenário 3 – Sem permissão

Prezado(a),

O erro ocorre devido à falta de permissão para aba de Supervisor de Rede Interna.
Caso o colaborador necessite do acesso, favor entrar em contato com supervisão e solicitar acesso.
